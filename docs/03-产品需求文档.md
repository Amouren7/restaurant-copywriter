# 餐饮文案助手 — 产品需求文档（PRD）

> 版本：v1.0 | 更新日期：2026-05-31

---

## 一、产品概述

**一句话描述**：一个手机打开就能用的网站，餐饮老板点3下出5条文案，复制发朋友圈。

**核心设计原则**：
1. **极简** — 40岁老板不看说明书就会用
2. **快速** — 从打开到拿到文案不超过30秒
3. **好看** — 在手机上看起来像个正经产品，不像个人小作坊

## 三、提示词模板设计

### 3.1 朋友圈场景

#### 场景：今日特价
```
系统提示词：
你是一位餐饮行业的朋友圈文案高手。请为一家{category}店写5条朋友圈文案。
文案要突出今日特价，有食欲感，口语化，像老板自己在发圈。

用户输入：
品类：{category}（如：快餐）
今日特价菜：{dish}（如：红烧排骨盖饭）
原价：{original_price}（如：22元）
现价：{current_price}（如：15元）
限量：{limit}（如：限50份）
补充信息：{extra}（可选）

要求：
1. 每条30-60字
2. 必须提到具体菜名和价格
3. 有紧迫感（限时限购）
4. 口语化，带emoji
5. 5条风格各异：实惠、品质、情感、场景、限时各一条
```

#### 场景：新品推荐
```
提示词模板类似，侧重点不同：
- 突出"新"和"好吃"
- 有食材/口感描写
- 可带优惠信息
```

#### 其余4个场景同理，每个场景一套独立提示词模板。

## 四、免费限制逻辑

### 4.1 实现方式
使用浏览器 **localStorage** 存储使用记录，无需后端数据库。

```javascript
// 数据结构
{
  "usage_date": "2026-05-31",    // 日期
  "usage_count": 2,              // 当天已用次数
  "activation_code": null         // 激活码（第二期使用）
}
```

### 4.2 判断逻辑
```javascript
function checkUsageLimit() {
  const today = new Date().toISOString().split('T')[0];
  const data = JSON.parse(localStorage.getItem('copywriter_usage') || '{}');

  // 如果有激活码，不限次
  if (data.activation_code) return { allowed: true, remaining: Infinity };

  // 如果日期不同，重置计数
  if (data.usage_date !== today) {
    localStorage.setItem('copywriter_usage', JSON.stringify({
      usage_date: today,
      usage_count: 0,
      activation_code: null
    }));
    return { allowed: true, remaining: 5 };
  }

  // 判断是否还有次数
  const remaining = 5 - data.usage_count;
  return {
    allowed: remaining > 0,
    remaining: remaining
  };
}
```

## 五、响应式设计要求

| 设备 | 要求 |
|------|------|
| 手机（375px-414px） | **主要适配**。卡片纵向排列，按钮全宽，字号16px+ |
| 平板（768px） | 卡片2列排列 |
| 桌面（1024px+） | 卡片3列排列，最大宽度800px居中 |

**核心原则**：手机端体验 > 一切。老板99%在手机上用。
