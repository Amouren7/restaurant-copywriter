## 三、后端架构

### 3.1 server.py 结构

```
server.py
├─ 配置区
│  ├─ DEEPSEEK_API_KEY (环境变量)
│  ├─ DEEPSEEK_BASE_URL
│  └─ FLASK_APP 配置
│
├─ 路由
│  ├─ GET  /           ← 返回 index.html
│  ├─ POST /api/generate   ← 调用DeepSeek生成文案
│  ├─ POST /api/verify     ← 验证激活码（二期）
│  └─ GET  /api/health     ← 健康检查
│
└─ 核心函数
   ├─ generate_copy(category, scene, params)
   │  └─ 从prompts.py获取模板 → 填充参数 → 调用DeepSeek API → 返回文案
   │
   └─ handle_error(e)
      └─ 统一错误处理
```

### 3.2 API接口定义

#### POST /api/generate

**请求**：
```json
{
  "category": "快餐",
  "scene": "daily_special",
  "params": {
    "dish": "红烧排骨盖饭",
    "extra": "今天炖了3小时"
  }
}
```

**响应（成功）**：
```json
{
  "success": true,
  "copies": [
    {"id": 1, "text": "中午不知道吃啥？红烧排骨盖饭今天只要15块！..."},
    {"id": 2, "text": "每天现炖的红烧排骨，不隔夜不冷冻..."},
    {"id": 3, "text": "..."},
    {"id": 4, "text": "..."},
    {"id": 5, "text": "..."}
  ]
}
```

**响应（失败）**：
```json
{
  "success": false,
  "error": "API调用失败，请稍后重试"
}
```

#### POST /api/verify（二期预留）

**请求**：
```json
{
  "code": "XXXX-XXXX-XXXX"
}
```

**响应**：
```json
{
  "valid": true,
  "plan": "monthly",
  "expires": "2026-07-01"
}
```

### 3.3 错误处理

| 错误类型 | HTTP状态码 | 处理方式 |
|---------|-----------|---------|
| DeepSeek API超时 | 504 | 返回"生成超时，请重试" |
| DeepSeek API错误 | 502 | 返回"服务暂时不可用" |
| 参数缺失 | 400 | 返回"请填写完整信息" |
| 激活码无效 | 403 | 返回"激活码无效或已过期" |
| 未知错误 | 500 | 返回"出了点问题，请稍后重试" |

## 七、安全考虑

| 风险 | 措施 |
|------|------|
| API密钥泄露 | 使用环境变量，不写在代码里 |
| 恶意刷接口 | 前端限制+后端限流（Flask-Limiter） |
| XSS攻击 | 前端输出做转义处理 |
| 内容合规 | 提示词中加入"不要生成违规内容"的约束 |
| DeepSeek API暴露 | 后端代理调用，前端不直接接触API |
