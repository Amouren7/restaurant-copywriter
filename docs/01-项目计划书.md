# 餐饮文案助手 — 项目计划书

> 版本：v1.0 | 更新日期：2026-05-31

---

## 一、项目概述

### 1.1 项目名称
**餐饮文案助手** — 面向餐饮老板的在线文案生成工具

### 1.2 项目定位
一个在线网站工具，帮助餐饮店老板快速生成朋友圈文案和外卖平台文案。
老板只需**点选3下**，10秒出稿，复制即用。

### 1.3 核心价值主张
> 每天花20分钟想朋友圈文案？这里10秒搞定。
> 一天不到7毛钱，省下来的时间多备两个菜。

### 1.4 目标用户
| 维度 | 描述 |
|------|------|
| 年龄 | 30-55岁 |
| 职业 | 小餐饮店老板（夫妻店、个体户） |
| 技术水平 | 会用微信，不会用复杂软件 |
| 核心痛点 | 每天要发朋友圈/更新外卖，但不会写、不想写、没时间写 |
| 使用场景 | 午餐前发朋友圈、上新品时写描述、节日做活动 |

### 1.5 竞品分析
| 竞品 | 价格 | 我们的优势 |
|------|------|-----------|
| 找人代写 | 5-20元/条 | 我们几乎免费 |
| 代运营公司 | 800-2000元/月 | 我们19.9元/月 |
| ChatGPT/DeepSeek | 免费但需要自己写提示词 | 我们点选即用，不需要写提示词 |
| 自己写 | 时间成本30分钟/天 | 我们10秒出稿 |

## 二、产品规划

### 2.1 三期规划

| 阶段 | 时间 | 目标 | 核心动作 |
|------|------|------|---------|
| **第一期：免费验证** | 第1-2周 | 验证需求，积累用户 | 上线免费版，每天5次免费使用 |
| **第二期：闲鱼变现** | 第3-4周 | 验证付费意愿 | 闲鱼卖激活码，实现半自动收入 |
| **第三期：全自动支付** | 月入1000+后 | 规模化收入 | 接入支付平台，全自动付费流程 |

### 2.2 第一期功能清单（MVP）

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 朋友圈文案生成 | P0 | 6个子场景：今日特价/新品/天气/好评/节日/日常 |
| 外卖文案生成 | P0 | 3个子场景：菜品标题/菜品描述/批量优化 |
| 一键复制 | P0 | 点击即复制到剪贴板 |
| 每日免费限制 | P0 | 每天5次，用localStorage记录 |
| 品类选择 | P0 | 9个常见品类：快餐/火锅/奶茶/烧烤等 |
| 移动端适配 | P0 | 手机端体验优先 |
| 用户反馈展示 | P1 | 上线后补充真实评价 |
| 激活码系统 | P1 | 预留接口，第二期启用 |
| 更多品类 | P2 | 根据用户需求扩展 |
| 抖音文案 | P2 | 二期扩展场景 |

### 2.3 不做的事（明确边界）
- ❌ 不做用户注册系统（localStorage足够）
- ❌ 不做AI对话聊天（只做点选式生成）
- ❌ 不做图片生成（只做文字文案）
- ❌ 不做多语言（只做中文）
- ❌ 不做APP（只做网页版）

## 三、技术方案

### 3.1 技术栈
| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | HTML + Tailwind CSS + Vanilla JS | 单文件，零框架依赖，加载快 |
| 后端 | Python Flask | 简单，50行搞定API |
| AI引擎 | DeepSeek API | 用户已有API key，成本低 |
| 部署 | Railway | 免费额度，自带域名 |
| 数据 | localStorage + JSON文件 | 第一期不需要数据库 |

### 3.2 系统架构
```
用户手机浏览器
    ↓ HTTPS
┌─────────────────────┐
│   Flask Server      │
│   ├─ index.html     │ ← 前端页面
│   ├─ /api/generate  │ ← 调用DeepSeek生成文案
│   └─ /api/verify    │ ← 验证激活码（预留）
└─────────────────────┘
    ↓ API调用
┌─────────────────────┐
│   DeepSeek API      │
└─────────────────────┘
```

### 3.3 数据流
```
用户点选品类+场景
    ↓
前端构造提示词（基于prompts.py模板）
    ↓
POST /api/generate {prompt, scene, category}
    ↓
后端调用DeepSeek API
    ↓
返回文案列表 [{text: "..."}, ...]
    ↓
前端展示 + 一键复制按钮
```

| 阶段 | 变现方式 | 单价 | 目标月收入 |
|------|---------|------|-----------|

## 五、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 没人用 | 中 | 高 | 小红书精准引流+免费策略降低门槛 |
| DeepSeek API收费涨价 | 低 | 中 | 备选：通义千问API，切换成本低 |
| 网站被封/域名被墙 | 低 | 高 | 用Railway自带域名，内容合规 |
| 用户觉得AI文案质量不行 | 中 | 中 | 持续优化提示词，提供"换一批"功能 |
| 同类产品竞争 | 中 | 低 | 先发优势+持续迭代+用户口碑 |
